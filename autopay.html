Forwarding AutoPay

Discord AutoPay without making sellers enter IMAP settings.

Premium servers can use email forwarding for PayPal and Cash App notifications. Store Bot matches the forwarded notification to the pending order code and amount.

How forwarding AutoPay works

  1. Customer orders from the panelStore Bot creates a unique order code and shows the amount due.
  2. Seller forwards payment notificationsThe seller forwards only PayPal or Cash App payment emails to their Store Bot forwarding address.
  3. Store Bot matches the orderThe bot checks the order code, amount, currency, provider, status, and server forwarding alias.
  4. Order is marked paidIf everything matches, Store Bot marks the order paid and can auto-deliver configured delivery text.
Premium AutoPay

No seller IMAP login required.

Forwarding mode avoids making every premium server owner enter IMAP settings. Store Bot gives each premium server forwarding instructions through /autopay forwarding.

Optional direct IMAP mode can still exist for users who prefer it, but the website now explains forwarding as the main AutoPay flow.

Matching checks

AutoPay is designed to avoid false matches.

Store Bot should only confirm payments when important fields match the pending order.

Order code

The forwarded payment notification must include the order code shown by Store Bot.

Amount and currency

The payment amount and currency must match the pending Discord order.

Provider

Forwarding AutoPay is built around PayPal and Cash App payment notification emails.

Status

The notification must indicate a received, completed, or paid transaction.

Server alias

Each premium server gets forwarding instructions so Store Bot can separate server payments.

Pending order

Store Bot only marks active pending orders paid; old, cancelled, or already-delivered orders are ignored.
