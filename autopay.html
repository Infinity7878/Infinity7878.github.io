AutoPay forwarding

Automatic order matching from forwarded payment emails.

Premium servers can forward PayPal or Cash App payment notification emails to Store Bot. The bot checks the forwarded message against the pending order before marking anything paid.

How it works

  1. Customer starts an orderStore Bot creates a pending order with an amount and order code.
  2. Seller receives paymentThe customer pays using the seller's configured payment method.
  3. Payment email is forwardedThe seller forwards the PayPal or Cash App notification email to the Store Bot forwarding address.
  4. Store Bot checks the detailsThe bot compares provider, order code, amount, currency, status, and pending order state.
  5. Matching orders are marked paidIf the details match, Store Bot can update the order and use configured delivery text.

Safer setup

No full inbox forwarding.

Only payment notification emails related to Store Bot orders should be forwarded. Store Bot does not need personal emails, unrelated receipts, or full inbox access for forwarding mode.

Matching checks

Designed to reduce false positives.

AutoPay should only confirm an order when the important fields line up.

Order code

The forwarded email should include the Store Bot order code.

Amount and currency

The payment total should match the pending order.

Provider

Forwarding mode is built around PayPal and Cash App payment notifications.

Status

The notification must indicate a received, completed, or paid transaction.

Server alias

Each premium server uses forwarding instructions that keep server payments separated.

Pending order

Old, cancelled, and already-delivered orders should not be matched again.
